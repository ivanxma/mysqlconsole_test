DBConsole MySQL Utility Refactor Plan

Plan to extract MySQL-specific connection, profile, TLS, cursor, transaction, and verification behavior from app.py into a dedicated modules/mysql_util.py module.

Target Modulemodules/mysql_util.py
Connectormysql-connector-python
Primary RiskConnection lifecycle and deployment parity
Deployment ScopemacOS, OL8, OL9, Ubuntu, OCI Compute

Goals

  • Move MySQL-specific API code out of app.py into a reusable utility module with a narrow, testable public API.
  • Keep existing behavior for saved profiles, SSH tunnels, Unix sockets, TLS modes, cached connections, SELECT 1 checks, and transaction handling.
  • Make Connector/Python behavior explicit and isolated so future driver changes do not spread across routes and page modules.
  • Preserve deployment behavior across setup.sh, start scripts, Auto-Update, and OCI first-boot setup.
  • Add validation coverage for local development, Linux systemd services, macOS local hosting, and OCI Compute initialization.

Non-Goals

  • Do not rewrite dashboard, DB Admin, HeatWave, Monitoring, or Import feature logic.
  • Do not change profile secrets storage or put database passwords in profiles.json.
  • Do not change the app-managed local MySQL provisioning model in setup.sh.
  • Do not introduce SQLAlchemy or another ORM.
  • Do not change OCI tenancy, compartment, subnet, or image assumptions beyond validation documentation.

Current Coupling To Remove

Area Current Location Problem Refactor Destination
Connector imports and exceptions app.py, modules/mysql_pages.py Driver-specific API is spread across routing and page logic. modules/mysql_util.py exposes connector errors and helper predicates.
Profile normalization and TLS options app.py Profile schema and connection arguments are mixed with Flask session code. normalize_profile(), normalize_mysql_ssl_mode(), and TLS argument builders move into utility module.
Cached connection lifecycle app.py Connection cache cleanup is interleaved with credential session cleanup. Utility owns cache entries; Flask code calls explicit cleanup hooks.
Connector/Python cursor adapter app.py Route layer knows dictionary cursor implementation details. MySQLConnectionAdapter in utility module.
Health check app.py SELECT 1 policy should be enforced at one connection boundary. test_connection() and prepare_connection_for_use().

Target Module API

Function or Class Responsibility Consumers
DEFAULT_PROFILE, MYSQL_SSL_MODES Define normalized connection profile defaults and valid TLS modes. app.py, profile routes, setup profile patch checks.

quote_sql_string_literal(value) Provide a driver-independent SQL literal helper for DB Admin comments. modules/mysql_pages.py.

Implementation Phases

Phase 1: Extract Pure Profile And TLS Helpers

  1. Create modules/mysql_util.py.
  2. Move DEFAULT_PROFILE, MYSQL_SSL_MODES, normalize_mysql_ssl_mode(), normalize_profile(), public_profile(), public_profiles(), and profile_signature().
  3. Update imports in app.py.
  4. Run compile checks before moving connection code.

Phase 2: Extract Connector Adapter

  1. Move MySQLConnectorConnection to modules/mysql_util.py.
  2. Rename it to MySQLConnectionAdapter for driver-neutral app semantics.
  3. Keep cursor(dictionary=True, buffered=True) inside the adapter.
  4. Keep commit(), rollback(), select_db(), and set_autocommit() behavior unchanged.

Phase 3: Extract Connection Cache Manager

  1. Move apply_mysql_ssl_profile(), open_mysql_connection(), test_mysql_connection(), prepare_mysql_connection_for_use(), and close_cached_mysql_connection().
  2. Replace app-level logic with borrow_connection() context manager.
  3. Keep session credential ownership in app.py; pass the session entry into the utility module.
  4. Keep SSH tunnel lifecycle attached to the same cache entry as the MySQL connection.

Phase 4: Normalize Exception Handling

  1. Expose OperationalError, InterfaceError, and ProgrammingError aliases from the utility module.
  2. Update app.py error handlers to import utility aliases.
  3. Update modules/mysql_pages.py to depend on utility aliases instead of mysql.connector directly.
  4. Keep error-code checks for missing table error 1146.

Phase 5: Move SQL Literal Helper

  1. Move DB Admin comment escaping to quote_sql_string_literal() in modules/mysql_util.py.
  2. Use a focused unit-style validation matrix for quotes, backslashes, NUL, newlines, carriage returns, and Ctrl-Z.
  3. Update modules/mysql_pages.py to import only the helper and exception aliases.

Phase 6: Documentation And Deployment Notes

  1. Update README.md to name modules/mysql_util.py as the driver boundary.
  2. Document Connector/Python dependency and profile TLS behavior.
  3. Document that setup-created deployments install Python 3.12+ and mysql-connector-python from requirements.txt.

Phase 7: Sync The myapp Skill

  1. Update /Users/ivanma/.codex/skills/myapp/SKILL.md so the default MySQL driver is mysql-connector-python, not PyMySQL.
  2. Add a generation rule that MySQL connection/profile/TLS/cache code belongs in modules/mysql_util.py for DBConsole-style apps instead of directly in app.py.
  3. Document the expected public API: profile normalization, public profile helpers, Connector/Python adapter, TLS kwargs, cached connection borrowing, SELECT 1 health checks, transaction cleanup, SSH tunnel cleanup, and SQL literal escaping.
  4. Update the skill’s default dependency guidance so new apps use mysql-connector-python>=9.5,<10.0 with sshtunnel only when SSH tunneling remains configurable.
  5. Add validation guidance to the skill: compile app.py and modules/*.py, run pip check, validate TLS kwargs, mock cached connection reuse, run bash -n on setup/start/OCI scripts, and check Connector/Python wheel availability for target platforms.

Regression Scenarios

Scenario Expected Behavior
Remote HeatWave or MySQL DB System with require_secure_transport=ON SSL Mode = Required connects through Connector/Python with TLS enabled; no error 3159.
Cached connection dies between page loads Next borrow runs SELECT 1, closes dead cache entry, opens a new connection, and retries once.
Profile changes after login Profile signature mismatch closes cached connection and opens a connection for the new profile settings.
Logout Server-side credentials, cached MySQL connection, and SSH tunnel are closed and removed.
Session expiry cleanup Expired server-side session cleanup closes cached connection and tunnel.
Local socket admin profile Utility uses Unix socket and never combines socket mode with SSH tunnel mode.
CSV/JSON import transaction failure Rollback runs, exception propagates, and cached connection autocommit is restored for later pages.

Rollback Plan

  1. Keep the refactor in one commit after validation passes.
  2. If runtime issues appear, revert only the refactor commit while retaining the already validated Connector/Python dependency change if needed.
  3. Clear any active server-side sessions by restarting DBConsole after rollback so cached connection objects are not reused across code versions.
  4. Rerun ./setup.sh <os_family> <deploy_mode> on deployed hosts if dependency files were changed.

Definition Of Done

  • Required modules/mysql_util.py owns connector imports, profile normalization, TLS kwargs, connection cache, health check, and SQL literal helper.
  • Required app.py keeps Flask routes, session ownership, and profile persistence only.
  • Required modules/mysql_pages.py no longer imports Connector/Python directly unless a narrow exception alias import is intentionally retained.
  • Required The local myapp skill is updated so future MySQL apps use mysql-connector-python and the modules/mysql_util.py refactor pattern.
  • Required Static Python, shell syntax, dependency, cache, TLS, socket, SSH, and transaction checks pass.
  • Required Platform validation evidence is captured for macOS and at least one Linux systemd deployment before release.
  • Release Gate OCI Compute validation is completed for OL9 and Ubuntu before marking OCI setup as fully verified.
